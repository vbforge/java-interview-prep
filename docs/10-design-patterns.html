Q87
Name the three main groups of design patterns

Short answer

The three main groups are Creational, Structural, and Behavioral patterns, as classified by the Gang of Four (GoF) in their seminal book "Design Patterns: Elements of Reusable Object-Oriented Software."

In depth

1. Creational Patterns — Object creation:

  • Abstract the instantiation process
  • Make system independent of how objects are created
  • Deal with object creation mechanisms
  • Examples: Singleton, Factory, Abstract Factory, Builder, Prototype

2. Structural Patterns — Object composition:

  • Concerned with how classes and objects are composed
  • Form larger structures from smaller parts
  • Focus on relationships between entities
  • Examples: Adapter, Bridge, Composite, Decorator, Facade, Flyweight, Proxy

3. Behavioral Patterns — Object communication:

  • Focus on communication between objects
  • Define how objects interact and distribute responsibility
  • Describe algorithms and flow of control
  • Examples: Chain of Responsibility, Command, Interpreter, Iterator, Mediator, Memento, Observer, State, Strategy, Template Method, Visitor

Q88
Name one pattern from each group

Short answer

Creational: Singleton — ensures only one instance of a class exists.
Structural: Proxy — provides a surrogate or placeholder for another object.
Behavioral: Strategy — defines a family of algorithms, makes them interchangeable.

In depth

1. Singleton (Creational):

// Thread-safe Singleton with double-checked locking
public class DatabaseConnection {
    private static volatile DatabaseConnection instance;
    
    private DatabaseConnection() { }
    
    public static DatabaseConnection getInstance() {
        if (instance == null) {
            synchronized (DatabaseConnection.class) {
                if (instance == null) {
                    instance = new DatabaseConnection();
                }
            }
        }
        return instance;
    }
}

// Alternative: Enum Singleton (most robust)
public enum Singleton {
    INSTANCE;
    public void doSomething() { ... }
}

2. Proxy (Structural):

// Proxy pattern - controls access to real object
interface Image {
    void display();
}

// Real object
class RealImage implements Image {
    private String filename;
    
    public RealImage(String filename) {
        this.filename = filename;
        loadFromDisk();
    }
    
    private void loadFromDisk() {
        System.out.println("Loading: " + filename);
    }
    
    @Override
    public void display() {
        System.out.println("Displaying: " + filename);
    }
}

// Proxy - lazy loading
class ProxyImage implements Image {
    private RealImage realImage;
    private String filename;
    
    public ProxyImage(String filename) {
        this.filename = filename;
    }
    
    @Override
    public void display() {
        if (realImage == null) {
            realImage = new RealImage(filename);  // Lazy load
        }
        realImage.display();
    }
}

3. Strategy (Behavioral):

// Strategy pattern - interchangeable algorithms
interface PaymentStrategy {
    void pay(int amount);
}

// Concrete strategies
class CreditCardPayment implements PaymentStrategy {
    @Override
    public void pay(int amount) {
        System.out.println("Paid " + amount + " with Credit Card");
    }
}

class PayPalPayment implements PaymentStrategy {
    @Override
    public void pay(int amount) {
        System.out.println("Paid " + amount + " with PayPal");
    }
}

// Context
class ShoppingCart {
    private PaymentStrategy paymentStrategy;
    
    public void setPaymentStrategy(PaymentStrategy strategy) {
        this.paymentStrategy = strategy;
    }
    
    public void checkout(int amount) {
        paymentStrategy.pay(amount);
    }
}

// Usage
ShoppingCart cart = new ShoppingCart();
cart.setPaymentStrategy(new CreditCardPayment());
cart.checkout(100);  // Paid 100 with Credit Card
cart.setPaymentStrategy(new PayPalPayment());
cart.checkout(50);   // Paid 50 with PayPal
// JUNIOR NOTE: In interviews, pick patterns you can explain well. Singleton is the easiest creational pattern to explain. Strategy is great for behavioral — it's intuitive and widely used. For structural, Proxy or Adapter are good choices that are easy to explain with real-world examples.
Q89
Which three patterns do you most often see in Spring?

Short answer

The three most common design patterns in Spring are Singleton (beans are singleton by default), Factory (BeanFactory and ApplicationContext), and Proxy (AOP, transaction management, lazy loading).

In depth

1. Singleton Pattern — Spring Beans:

  • Where: All Spring beans are singletons by default
  • How: Spring container creates exactly one instance per bean definition
  • Why: Stateless services, repositories, controllers — no need for multiple instances
// Spring's default singleton scope
@Service
public class UserService {
    // One instance per application context
    // Spring ensures only one instance exists
}

2. Factory Pattern — BeanFactory & ApplicationContext:

  • Where: BeanFactory and ApplicationContext interfaces
  • How: Spring acts as a factory that creates and manages beans
  • Why: Abstract the bean creation process, support different creation strategies
// Spring's Factory Pattern in action
@Configuration
public class AppConfig {
    @Bean
    public DataSource dataSource() {
        // Factory method that creates a bean
        return DataSourceBuilder.create().build();
    }
    
    @Bean
    public UserService userService() {
        // Factory creates beans with dependencies
        return new UserService(userRepository());
    }
}

3. Proxy Pattern — AOP, Transactions, Lazy Loading:

  • Where: Transaction management, security, caching, lazy loading
  • How: Spring creates dynamic proxies around beans (JDK proxy or CGLIB)
  • Why: Add cross-cutting concerns without modifying business logic
// Spring's Proxy Pattern with @Transactional
@Service
public class OrderService {
    @Transactional  // Spring creates a proxy for transaction management
    public void createOrder(Order order) {
        // Business logic
        // Transaction is managed by the proxy
    }
}

📋 Quick-reference cheat sheet
GoF Pattern Groups:

  Creational (object creation):
    • Singleton → one instance
    • Factory → create objects without exposing logic
    • Builder → complex object construction
    • Prototype → clone objects
    • Abstract Factory → families of objects

  Structural (composition):
    • Proxy → placeholder for another object
    • Adapter → incompatible interfaces
    • Decorator → add responsibilities dynamically
    • Facade → simplified interface
    • Composite → tree structures
    • Bridge → abstraction from implementation
    • Flyweight → share objects efficiently

  Behavioral (communication):
    • Strategy → interchangeable algorithms
    • Observer → publish/subscribe
    • Command → encapsulate requests
    • Chain of Responsibility → pipeline processing
    • Iterator → traverse collections
    • Template Method → define algorithm skeleton
    • State → behavior based on state
    • Mediator → centralize communication
    • Memento → capture/restore state
    • Visitor → separate algorithm from structure

Spring Patterns:
  Singleton   → beans are singleton by default
  Factory     → BeanFactory, ApplicationContext
  Proxy       → AOP, Transactions, Lazy Loading
  Template    → JdbcTemplate, RestTemplate
  Front Controller → DispatcherServlet
  Observer    → ApplicationEvent, @EventListener
  Strategy    → View resolvers, MessageConverters

  ✅ Always mention Singleton + Factory + Proxy in Spring interviews!
